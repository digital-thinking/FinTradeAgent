================================================================================
LANGGRAPH MULTI-AGENT DEBATE SYSTEM - CONCEPT & IMPLEMENTATION PLAN
================================================================================

OVERVIEW
--------
Migrate from simple single-shot LLM calls to LangGraph-based agent orchestration.
This enables:
1. Structured agent workflows with state management
2. Multi-agent debates between bull/bear personas
3. Tool use (web search, price lookup) as graph nodes
4. Retry/error handling built into the graph
5. Streaming and observability


================================================================================
PART 1: MIGRATE EXISTING SIMPLE AGENT TO LANGGRAPH
================================================================================

CURRENT ARCHITECTURE
--------------------
```
AgentService.execute()
    └── _build_prompt()
    └── _invoke_openai() or _invoke_anthropic()
    └── _parse_response()
    └── return AgentRecommendation
```

Single LLM call with web search enabled. No state management between steps.


PROPOSED LANGGRAPH ARCHITECTURE
-------------------------------
```
                    ┌─────────────┐
                    │   START     │
                    └──────┬──────┘
                           │
                    ┌──────▼──────┐
                    │  Research   │  (web search for market data)
                    │    Node     │
                    └──────┬──────┘
                           │
                    ┌──────▼──────┐
                    │  Analysis   │  (apply strategy logic)
                    │    Node     │
                    └──────┬──────┘
                           │
                    ┌──────▼──────┐
                    │  Generate   │  (create trade recommendations)
                    │   Trades    │
                    └──────┬──────┘
                           │
                    ┌──────▼──────┐
                    │  Validate   │  (check JSON format, constraints)
                    │    Node     │
                    └──────┬──────┘
                           │
              ┌────────────┼────────────┐
              │ valid      │            │ invalid
              ▼            │            ▼
        ┌─────────┐        │      ┌─────────┐
        │   END   │        │      │  Retry  │──── (max 3 retries)
        └─────────┘        │      └─────────┘
                           │
                    (loops back to Generate)
```


STATE SCHEMA
------------
```python
from typing import TypedDict, Annotated
from langgraph.graph.message import add_messages

class SimpleAgentState(TypedDict):
    # Input
    portfolio_config: PortfolioConfig
    portfolio_state: PortfolioState

    # Research results
    market_research: str  # web search results
    price_data: dict[str, float]  # ticker -> current price

    # Analysis
    analysis: str  # strategy-specific analysis

    # Output
    messages: Annotated[list, add_messages]
    recommendations: AgentRecommendation | None

    # Control
    retry_count: int
    error: str | None
```


NODE IMPLEMENTATIONS
--------------------

1. research_node:
   - Takes portfolio context (holdings, strategy)
   - Calls LLM with web search to gather relevant market data
   - Stores results in state.market_research

2. analysis_node:
   - Takes research results + strategy prompt
   - Applies strategy logic to identify opportunities
   - Stores analysis in state.analysis

3. generate_trades_node:
   - Takes analysis + portfolio constraints
   - Generates JSON trade recommendations
   - Stores in state.recommendations (or error if parse fails)

4. validate_node:
   - Checks JSON structure
   - Validates against constraints (cash, holdings)
   - Routes to END or retry


CONDITIONAL EDGES
-----------------
```python
def should_retry(state: SimpleAgentState) -> str:
    if state["error"] and state["retry_count"] < 3:
        return "retry"
    elif state["recommendations"]:
        return "end"
    else:
        return "end"  # give up after max retries
```

================================================================================
PART 2: MULTI-AGENT DEBATE SYSTEM
================================================================================

CONCEPT
-------
Instead of one agent making decisions, we simulate an investment committee:

1. BULL AGENT: Finds reasons to buy, focuses on upside
2. BEAR AGENT: Finds reasons to sell/avoid, focuses on risks
3. MODERATOR: Synthesizes debate, makes final decision

Each agent has access to the same tools (web search, prices) but different
prompts that bias their analysis.


DEBATE FLOW
-----------
```
                         ┌─────────────┐
                         │   START     │
                         └──────┬──────┘
                                │
                         ┌──────▼──────┐
                         │  Research   │  (shared research phase)
                         │   Phase     │
                         └──────┬──────┘
                                │
              ┌─────────────────┼─────────────────┐
              │                 │                 │
       ┌──────▼──────┐   ┌──────▼──────┐   ┌──────▼──────┐
       │    BULL     │   │    BEAR     │   │  NEUTRAL    │
       │   Agent     │   │   Agent     │   │   Agent     │
       │  (Pitch)    │   │  (Pitch)    │   │  (Pitch)    │
       └──────┬──────┘   └──────┬──────┘   └──────┬──────┘
              │                 │                 │
              └─────────────────┼─────────────────┘
                                │
                         ┌──────▼──────┐
                         │   DEBATE    │  (agents respond to each other)
                         │   Round 1   │
                         └──────┬──────┘
                                │
                         ┌──────▼──────┐
                         │   DEBATE    │  (rebuttals)
                         │   Round 2   │
                         └──────┬──────┘
                                │
                         ┌──────▼──────┐
                         │  MODERATOR  │  (synthesize + decide)
                         │   Verdict   │
                         └──────┬──────┘
                                │
                         ┌──────▼──────┐
                         │  Generate   │  (final recommendations)
                         │   Trades    │
                         └──────┬──────┘
                                │
                         ┌──────▼──────┐
                         │    END      │
                         └─────────────┘
```


DEBATE STATE SCHEMA
-------------------
```python
class DebateAgentState(TypedDict):
    # Input
    portfolio_config: PortfolioConfig
    portfolio_state: PortfolioState

    # Shared research
    market_research: str
    price_data: dict[str, float]

    # Agent pitches (round 0)
    bull_pitch: str
    bear_pitch: str
    neutral_pitch: str

    # Debate rounds
    debate_history: list[dict]  # [{agent, message, round}]
    current_round: int
    max_rounds: int

    # Moderator
    moderator_analysis: str
    final_verdict: str  # "BUY", "SELL", "HOLD" with reasoning

    # Output
    recommendations: AgentRecommendation | None

    # Metadata
    messages: Annotated[list, add_messages]
```


AGENT PERSONAS
--------------

BULL AGENT PROMPT:
```
You are the BULL on an investment committee. Your job is to find compelling
reasons to BUY. Focus on:
- Growth potential and TAM expansion
- Competitive advantages and moats
- Positive catalysts on the horizon
- Undervaluation relative to peers
- Management execution and vision

Be aggressive but not reckless. Back claims with data from your research.
You must advocate for the long position.
```

BEAR AGENT PROMPT:
```
You are the BEAR on an investment committee. Your job is to find reasons
NOT to buy or to SELL. Focus on:
- Valuation concerns and downside risk
- Competitive threats and disruption risk
- Accounting red flags or aggressive assumptions
- Macro headwinds and sector rotation
- Management credibility issues

Be thorough but not paranoid. Back concerns with specific evidence.
Your job is to stress-test every idea.
```

NEUTRAL AGENT PROMPT:
```
You are the NEUTRAL ANALYST on an investment committee. Your job is to
provide balanced, objective analysis. Focus on:
- Fair value estimation using multiple methods
- Risk/reward asymmetry assessment
- Position sizing considerations
- Time horizon alignment with strategy
- Key metrics to monitor

Don't pick sides. Present the facts and tradeoffs clearly.
```

MODERATOR PROMPT:
```
You are the CIO moderating this investment committee debate. You've heard:
- Bull case: {bull_arguments}
- Bear case: {bear_arguments}
- Neutral analysis: {neutral_analysis}

Synthesize the debate and make a final decision. Consider:
1. Which arguments are most compelling and why?
2. What's the risk/reward asymmetry?
3. How does this fit the portfolio strategy?
4. What position size is appropriate given conviction?

Deliver a clear verdict: BUY, SELL, or HOLD with specific reasoning.
```


PARALLEL EXECUTION
------------------
LangGraph supports parallel node execution. The three agent pitches can
run simultaneously:

```python
from langgraph.graph import StateGraph

graph = StateGraph(DebateAgentState)

# Parallel pitch phase
graph.add_node("bull_pitch", bull_pitch_node)
graph.add_node("bear_pitch", bear_pitch_node)
graph.add_node("neutral_pitch", neutral_pitch_node)

# Fan out from research to all three
graph.add_edge("research", "bull_pitch")
graph.add_edge("research", "bear_pitch")
graph.add_edge("research", "neutral_pitch")

# Fan in to debate (waits for all three)
graph.add_edge("bull_pitch", "debate_round")
graph.add_edge("bear_pitch", "debate_round")
graph.add_edge("neutral_pitch", "debate_round")
```


DEBATE ROUNDS
-------------
Each debate round:
1. Each agent sees the other agents' previous statements
2. Each agent responds/rebuts
3. Responses added to debate_history
4. Continue until max_rounds reached

```python
def debate_round_node(state: DebateAgentState) -> DebateAgentState:
    round_num = state["current_round"]

    # Get previous round's statements
    prev_statements = [
        msg for msg in state["debate_history"]
        if msg["round"] == round_num - 1
    ]

    # Each agent responds (could parallelize)
    bull_response = invoke_bull_rebuttal(prev_statements)
    bear_response = invoke_bear_rebuttal(prev_statements)

    # Add to history
    state["debate_history"].extend([
        {"agent": "bull", "message": bull_response, "round": round_num},
        {"agent": "bear", "message": bear_response, "round": round_num},
    ])

    state["current_round"] += 1
    return state
```

================================================================================
OPEN QUESTIONS
================================================================================

1. Should debate agents have access to DIFFERENT data sources?
   - Bull only sees positive news?
   - Or same data, different interpretation?
   → Recommendation: Same data, different prompts

2. How many debate rounds are optimal?
   - More rounds = more tokens = higher cost
   - Diminishing returns after 2-3 rounds?
   → Start with 2, make configurable

3. Should we weight agent opinions differently?
   - Track historical accuracy per agent persona?
   - Adjust moderator weighting based on track record?
   → Future enhancement

4. How to handle agent disagreement on facts?
   - Bull says revenue grew, Bear says it didn't
   - Need fact-checking step?
   → Add validation node that checks claims

5. Cost management:
   - Debate uses 4-5x more tokens than simple
   - Worth it for high-conviction decisions only?
   → Add cost estimate before running, let user choose
